# GAS + GPT セットアップ手順

## 概要

この手順で、れんの恋愛タロットに GPT 連携を追加します。

```
[ユーザー] → [GitHub Pages] → [GAS] → [OpenAI API] → [鑑定文]
```

---

## Step 1: OpenAI API キーを取得

1. https://platform.openai.com/ にログイン
2. 右上のアイコン → **API keys**
3. **Create new secret key** をクリック
4. 名前を入力（例: `ren-tarot`）して作成
5. 表示されたキー（`sk-...`）をコピーして保存
   - ⚠️ このキーは一度しか表示されない

---

## Step 2: Google Apps Script プロジェクト作成

1. https://script.google.com/ にアクセス
2. **新しいプロジェクト** をクリック
3. プロジェクト名を「れんタロットAPI」などに変更
4. デフォルトの `コード.gs` の中身を全部消す
5. `gas/Code.gs` の内容をコピペ
6. **保存**（Ctrl+S）

---

## Step 3: API キーを設定

1. 左メニューの **歯車アイコン（プロジェクトの設定）** をクリック
2. 下にスクロールして **スクリプト プロパティ** セクションへ
3. **スクリプト プロパティを追加** をクリック
4. 以下を入力:
   - プロパティ: `OPENAI_API_KEY`
   - 値: Step 1 でコピーしたキー
5. **スクリプト プロパティを保存**

---

## Step 4: テスト実行

1. エディタに戻る
2. 関数選択で `testGenerateReading` を選ぶ
3. **実行** をクリック
4. 初回は認証を求められる → **権限を確認** → アカウント選択 → **許可**
5. 下の **実行ログ** に鑑定文が表示されれば成功

---

## Step 5: ウェブアプリとしてデプロイ

1. 右上の **デプロイ** → **新しいデプロイ**
2. **種類の選択** → 歯車アイコン → **ウェブアプリ**
3. 設定:
   - 説明: `v1`（なんでもOK）
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**
4. **デプロイ** をクリック
5. **ウェブアプリの URL** をコピー
   - 例: `https://script.google.com/macros/s/AKfyc.../exec`

---

## Step 6: フロントエンドに URL を設定

1. `assets/js/reading.js` を開く
2. 15行目あたりの `GAS_API_URL` を書き換え:

```javascript
const GAS_API_URL = "https://script.google.com/macros/s/あなたのURL/exec";
```

3. 保存
4. GitHub Desktop でコミット & プッシュ

---

## Step 7: 動作確認

1. https://kasu-create.github.io/paid-oracle-site/oracle/reading/ を開く
2. 悩みと生年月日を入力
3. カードを引く
4. 「れんの鑑定を読む」をクリック
5. ローディング後、GPT が生成した鑑定文が表示されれば成功！

---

## トラブルシューティング

### 鑑定文が表示されない

1. ブラウザの開発者ツール（F12）→ Console を確認
2. エラーが出ていたら内容をチェック

### CORS エラーが出る

GAS のウェブアプリは自動で CORS 対応されるので、通常は出ない。
出る場合は URL が間違っている可能性が高い。

### API キーエラー

1. GAS のプロジェクト設定を開く
2. スクリプトプロパティで `OPENAI_API_KEY` が正しく設定されているか確認
3. キーが失効していないか OpenAI ダッシュボードで確認

### GPT の応答が遅い

- 初回は 10〜20 秒かかることがある
- `gpt-4o-mini` を使っているので比較的速いはず
- GAS のタイムアウトは 30 秒
